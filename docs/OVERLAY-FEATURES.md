# Counterplay — реализованный функционал (этап 5: WPF-оверлей)

Документ описывает всё, что добавлено **после коммита `e762561`** ("Add all
champions, real data pipeline export…"). На этом этапе консольный коннектор
превращён в полноценный **WPF-оверлей** поверх клиента LoL с живыми
рекомендациями, объяснениями связок и визуальным оформлением в стиле клиента.

## 1. Переход на WPF-оверлей

- `Counterplay.csproj` переведён на `net8.0-windows`, `WinExe`, `UseWPF`.
- `Program.cs` — точка входа `[STAThread]`, поднимает `Application` + `OverlayWindow`,
  LCU-цикл крутится в фоновом `Task`, события прокидываются в UI через `Dispatcher`.
- `AttachConsole(-1)` — WinExe цепляется к родительскому терминалу, чтобы работал Ctrl+C.
- Внешний цикл переподключения: ждёт lockfile, переживает перезапуск клиента,
  ждёт готовности LCU (`/lol-gameflow/v1/availability`).
- `GlobalUsings.cs` — глобальные using'и (WPF SDK не подключает System.IO/Net.Http сам).

## 2. Окно оверлея (`OverlayWindow.xaml` / `.xaml.cs`)

- Прозрачное, без рамки, always-on-top, перетаскиваемое; ресайз за любой край/угол
  (Win32 `WM_SYSCOMMAND`/`SC_SIZE`).
- **Два режима**: полный (3 колонки) и компактный; переключение кнопкой ⊟/⊞.
- **Экран ожидания**: маленькое окно со стадией загрузки крупным шрифтом и
  тремя поочерёдно мигающими точками (вместо пустоты, когда нет драфта).
- **Привязка к клиенту**: при появлении левый верхний угол оверлея встаёт к
  правому верхнему углу окна клиента LoL (Win32 `EnumWindows`, учёт DPI).
  Дальше окно свободно перетаскивается; кнопка 📌 возвращает к краю.

## 3. Иконки

- `IconCache.cs` — предзагрузка всех ~170 иконок чемпионов при старте
  (параллельно, 16 потоков), хранение байтов, создание `BitmapImage` из памяти
  (синхронно и надёжно `Freeze`-абельно).
- `RoleIcons.cs` — иконки ролей (топ/лес/мид/бот/саппорт) из Community Dragon;
  показываются в пустом слоте локального игрока.
- `DataDragon.GetAllIconUrls()` — список всех URL иконок для предзагрузки.

## 4. Карточки рекомендаций

- Раскладка: слева ранг (отдельный шрифт `NumberFont`) + очки, правее иконка,
  затем имя + WR + развёрнутое описание (фикс. ширина), справа — блок показателей.
- 4 показателя понятными словами: **Винрейт в патче**, **Против оппонента**,
  **Против вражеской команды**, **Синергия с союзниками** — с полосками и значениями.
- Очки (score) = взвешенная сумма вкладов (не винрейт).

## 5. Команды (союзники / враги)

- Иконки 72px, роль сверху + имя; разделители между чемпионами как в клиенте.
- **Подсветка моего слота**: яркая золотая рамка + свечение + иконка роли в пустом круге.
- **Боковые подсказки**: под врагами «КОНТРЫ», под союзниками «СИНЕРГИЯ» — мини-иконки
  лучших чемпионов (из БД, `TopCounters` / `TopSynergies`).

## 6. Командные связки (`TeamSynergies.cs`)

- Курируемая база комбо внутри состава: подброс→ульт Ясуо/Йоне, шаровая молния
  (Орианна), бессмертие (Тарик/Зилеан/Кайл + дайв), разгон гиперкэрри (энчантер+кэрри),
  пик через хук, тимфайт-ульты, глобальное давление.
- Карточки «СВЯЗКИ КОМАНДЫ / ВРАГОВ» под списком: иконки участников, описание и
  подсказка («КАК ИГРАТЬ» / «ОПАСНОСТЬ»).
- **Линии-коннекторы**: на Canvas рисуются скобки между портретами участников связки,
  цвет совпадает с цветной меткой карточки-легенды.
- `ExplainPair()` — развёрнутые объяснения «почему синергия» простым языком,
  с дедупликацией по типу связки (без повторов).

## 7. Движок рекомендаций (`RecommendationEngine.cs`)

- Учёт **ховеров** (`EffectiveChampionId`): рекомендации обновляются ещё на наведении.
- Отдельное сглаживание парных таблиц `K_PAIR=20` (вместо K=50 — данные синергии/
  матчапов очень разрежены).
- **Пул синергии/матчапов** по союзникам/врагам вместо усреднения с нулями;
  симметрия пары + маржинализация роли (`RawSynergy`) — максимум сигнала.
- Динамический вес синергии: без информации о врагах синергия важна как контрпик.
- `FindDb()` валидирует БД (есть таблица `base_wr`) — пустой `data.db` больше не перехватывает.
- Боковые подсказки с низким порогом `HINT_MIN_GAMES=5`.

## 8. Стабильность LCU (`LcuEventSocket.cs`, `OverlayWindow`)

- **Неблокирующий** `Dispatcher.InvokeAsync` для обновлений UI — иначе при лавине
  событий (переход в игру) поток чтения WebSocket вставал, буфер клиента переполнялся
  и клиент зависал («reconnect», игра не стартовала).
- Дешёвый префильтр сообщений по подстроке URI до полного JSON-парсинга.

## 9. Оформление в стиле клиента

- Шрифты: `DisplayFont` (Beaufort/Spiegel-подобный для имён), `NumberFont` для цифр.
- **Декоративный фон**: диагональная подсветка, тёплое/холодное угловые свечения,
  плавно дрейфующие «потоки» (бледные золотые кривые) — полупрозрачно, не мешает тексту.
- Дисклеймер «Not endorsed by Riot Games».

## Соблюдение ограничений Riot

- Подсказки только в фазе драфта.
- Имена врагов обфусцируются («ENEMY N»).
- Никакого чтения памяти игры — только LCU API.
- Свой пайплайн на MATCH-V5, без парсинга конкурентов.
