## Что строим
Десктоп-приложение под Windows для **контрпиков в League of Legends в реальном
времени** (как Porofessor/Blitz): определяет фазу драфта через LCU API, по составу
союзников/врагов рекомендует пик (контрпик) на основе винрейтов матчапов и
синергии, показывает сборку/руны/спеллы/порядок прокачки, опционально импортирует
руны в клиент одной кнопкой.

## Архитектура и стек
Принцип: тяжёлый сбор данных (batch, раз в патч) отделён от лёгкого рантайма —
оверлей просто читает готовые таблицы и отвечает мгновенно.
1. **LCU-коннектор** — lockfile, HTTP Basic (`riot:пароль` в base64), WebSocket WAMP. [C#/.NET 8]
2. **Парсер драфта** — myTeam/theirTeam, роли, прямой оппонент, баны; частичный драфт.
3. **Статика** — Data Dragon / Community Dragon (id → имя/иконка).
4. **Пайплайн данных (офлайн)** — MATCH-V5 → таблицы: базовый WR / контр-матчап /
   синергия (по роли/патчу/эло) в SQLite. [Python: riotwatcher/cassiopeia]
5. **Движок скоринга** — взвешенная сумма (база + контра против прямого оппонента
   [МАКС. вес] + средняя контра против прочих врагов + средняя синергия с
   союзниками); на **дельтах**, а не сырых WR; сглаживание малых выборок (нижняя
   граница Уилсона или Лаплас приор 0.5, k≈50; ниже ~50–100 игр — откат на базу).
6. **Оверлей** — прозрачное click-through always-on-top окно поверх клиента. [C#/WPF]
7. **Импорт рун** — опц., через LCU (`POST /lol-perks/v1/pages`).

## Что уже сделано (рабочее, проверено сборкой)

### A. C#-консольный коннектор — папка `C:\Counterplay` (этапы 1–2 готовы)
Читает lockfile, подключается к LCU по WebSocket (подписка `[5,"OnJsonApiEvent"]`,
события приходят как `[8,"OnJsonApiEvent",{uri,eventType,data}]`, фильтрую по `uri`,
самоподписанный серт localhost принимается), в реальном времени разбирает champ
select и печатает в консоль: фазы геймфлоу, мою роль, союзников, врагов, ховеры
(`championPickIntent`), баны. Подключён Data Dragon — имена чемпионов по-русски,
URL иконок готовы для оверлея. .NET 8, **без внешних NuGet-пакетов**. Собирается
чисто (`dotnet build`, 0 ошибок).

Файлы: `Program.cs` (точка входа, цикл событий), `Lcu.cs` (creds из lockfile +
HTTP-клиент к LCU), `LcuEventSocket.cs` (WebSocket/WAMP, поток событий),
`ChampSelect.cs` (модель драфта + парсер сессии), `DataDragon.cs` (id→имя/иконка),
`DraftPrinter.cs` (вывод в консоль — на этапе 5 заменится оверлеем),
`Counterplay.csproj`, `README.md`.
Запуск: `cd C:\Counterplay` → `dotnet run` (путь к lockfile уже стоит дефолтным;
переопределить можно аргументом).

### B. Веб-прототип «тестовый драфт» — папка `C:\Counterplay\test-draft` (только что собран)
Самодостаточный HTML/CSS/JS (без сборки и зависимостей) — песочница, чтобы
обкатать UI и движок рекомендаций **без** подключения к клиенту. Сам расставляешь
пики/баны (клик по слоту → клик по чемпиону в сетке), движок в реальном времени
выдаёт топ-6 на мою роль с текстовым обоснованием и полосками вклада (база / vs
оппонента / vs прочих / синергия). Вид под драфт LoL: союзники слева, враги справа,
рекомендации по центру. Тёмно-синяя тема + hextech-золото.

Скоринг как в плане: веса base 1.0, прямой контр 2.5, прочие контры 0.8, синергия
1.2; на дельтах. **Данные матчапов/синергии/WR — СИНТЕТИЧЕСКИЕ заглушки**
(детерминированно из id чемпионов, FNV-1a), чтобы видеть работу механики. Реальные
числа придут из пайплайна (этап 3, ещё не сделан). `score` — относительная оценка
(сумма дельт), не винрейт; важен порядок и разбивка.

Файлы: `index.html`, `styles.css`, `engine.js` (чемпионы + генерация данных +
скоринг + обоснование; написан так, что работает и в браузере, и в Node), `app.js`
(интерфейс/состояние/рендер/иконки), `README.md`.
Запуск: открыть `index.html` двойным кликом или через Live Server в VS Code.

### Почему две кодовые базы
C# — это **боевая** часть (коннектор + будущий WPF-оверлей), как в ТЗ. Веб-прототип —
быстрый одноразовый стенд, чтобы проверить UX и логику скоринга, не дожидаясь
пайплайна и не возясь с WPF. Логика скоринга из `engine.js` потом переносится в
боевой движок. Эти две папки между собой кодом не связаны.

## Технические находки / грабли
- В ранкеде Solo/Duo и в Blind Pick роли врага НЕ раскрываются (`assignedPosition`
  у `theirTeam` пуст) → «прямого оппонента на линии» по позиции не вычислить; надо
  выводить эвристикой «чемпион → частая роль» (из таблиц пайплайна). В Blind Pick
  враги к тому же скрыты целиком до загрузки игры, и банов нет.
- Champ select session: `myTeam`/`theirTeam` (поля `championId`, `assignedPosition`,
  `championPickIntent`=ховер, `cellId`), `localPlayerCellId`=мой слот, `actions[]`
  для банов (`type:"ban"`, `completed:true`, `isAllyAction`).
- Data Dragon: `versions.json[0]` = последний патч; в `champion.json` объект `data`
  ключуется по имени чемпиона, у каждого: `key`=числовой championId (строкой, как в
  LCU), `id`=английский идентификатор (для иконок), `name`=локализованное имя.
  Иконка: `cdn/{version}/img/champion/{id}.png`.
