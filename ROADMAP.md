# Counterplay — Roadmap

**Цель:** Windows-приложение, которое пользователь скачивает с сайта, устанавливает
один раз — и при каждом заходе в чемп-селект видит окно с рекомендациями контрпиков.

## Что уже готово ✅

| Этап | Что сделано |
|------|-------------|
| 1. LCU-коннектор | Lockfile → HTTP Basic + WebSocket WAMP; подписка на события |

## Что делаем дальше

### Этап 3 — Python-пайплайн данных  `следующий`
**Результат:** файл `data.db` (SQLite) с реальными матчап-статами.

**Решено:**
- Регион: **EUW** (`europe` для MATCH-V5, `euw1` для League-V4)
- Эло по умолчанию: **Emerald** (пользователь указал свой ранг)
- Авто-определение ранга: при запуске читаем `GET /lol-ranked/v1/current-ranked-stats`
  → поле `queueMap.RANKED_SOLO_5x5.tier` → выбираем нужный бакет данных

**Бакеты данных в data.db** (одна база на всех):

| Бакет | Кто использует |
|-------|----------------|
| `silver` | Iron–Silver |
| `gold` | Gold–Platinum |
| `emerald` | Emerald–Diamond |
| `master` | Master+ |

**Схема пайплайна:**
1. `collect.py` — League-V4: берём пул summonerIds по бакету/тиеру →
   MATCH-V5: matchlist по puuid → данные матча → агрегируем на лету
2. `aggregate.py` — считаем: base_wr / matchup (counter-delta) / synergy
   по (champion, role, vs_champion, vs_role, tier_bucket, patch)
3. Сглаживание: Лаплас-приор k=50, если игр < 100 — откат на базовый WR
4. Сырой JSON **не хранить** — только агрегат

**Таблицы SQLite:**
```sql
base_wr  (champion_id, role, tier_bucket, patch, games, wins)
matchup  (champion_id, role, vs_champion_id, vs_role, tier_bucket, patch, games, wins)
synergy  (champion_id, role, ally_id, ally_role, tier_bucket, patch, games, wins)
```

- Регистрация приложения на developer.riotgames.com (dev-ключ: 20 rps, 24 ч)
- Сырой JSON **не хранить** — только агрегат

### Этап 4 — C# движок скоринга
**Результат:** консоль выдаёт реальные рекомендации вместо заглушек.

- `RecommendationEngine.cs` читает `data.db` через Microsoft.Data.Sqlite
- Та же формула: Σ (база·1.0 + прямой контр·2.5 + прочие контры·0.8 + синергия·1.2)
- Эвристика «чемпион → роль» для Solo/Duo (где роли врага скрыты)
- `DraftPrinter.cs` выводит топ-4 с обоснованием (уже в консоль, потом уйдёт в оверлей)

---

### Этап 5 — WPF-оверлей  `ключевой этап`
**Результат:** окно поверх клиента, обновляется в реальном времени.

- Новый WPF-проект в решении, переиспользует коннектор и движок
- Окно: always-on-top, прозрачный фон, click-through (не мешает кликать в клиент)
- Позиция: справа от клиента или поверх по центру (уточним при разработке)
- Показывает: топ-4 рекомендации, иконки + имя + score + 2–3 строки обоснования
- Анимация обновления при каждом событии champ select
- Тёмная тема под LoL (как в веб-прототипе)

---

### Этап 6 — Импорт рун (опциональный)
**Результат:** кнопка «Импорт рун» в оверлее.

- `POST /lol-perks/v1/pages` через уже готовый `LcuHttpClient`
- Данные рун берём из Community Dragon (бесплатно, без API-ключа)
- Одна кнопка — импортирует руны первой рекомендации одним кликом

---

### Этап 7 — Сборка и дистрибуция  `финал`
**Результат:** пользователь скачивает `Counterplay-Setup.exe` и нажимает «Далее».

**7а. Упаковка приложения**
- `dotnet publish --self-contained -r win-x64` → один exe без зависимостей .NET
- Рядом кладём `data.db` (обновляется раз в патч)
- Опц.: авто-проверка нового патча при запуске → скачивает свежий `data.db`

**7б. Инсталлятор**
- [Inno Setup](https://jrsoftware.org/isinfo.php) — бесплатно, один `.iss`-скрипт
- Устанавливает в `%LOCALAPPDATA%\Counterplay`, опц. ярлык на рабочем столе
- Авто-запуск с Windows через реестр (опц., по желанию пользователя)

**7в. Дистрибуция**
- Первый релиз: GitHub Releases (бесплатно, надёжно)
- Дальше опц.: лендинг-страница с кнопкой Download
- **Важно:** SmartScreen покажет предупреждение при первом запуске у новых пользователей.
  Чтобы убрать — нужна подпись кода (Code Signing Certificate, ~$100–300/год).
  Альтернатива: репутация нарабатывается сама со временем (сотни скачиваний).

## Итоговая картина

```
Пользователь запускает Counterplay-Setup.exe
  → Устанавливается в %LOCALAPPDATA%\Counterplay
  → Запускается фоном при старте Windows (опц.)

Пользователь заходит в LoL → ищет матч
  → Counterplay читает lockfile, подключается к LCU
  → При входе в чемп-селект появляется оверлей
  → В реальном времени обновляет топ-4 рекомендации
  → При желании: кнопка «Импорт рун»
  → Чемп-селект заканчивается → оверлей скрывается
```

---

*Синтетические данные → реальные: только этап 3.*
*Консоль → оверлей: только этап 5.*
*Инструмент → продукт: только этап 7.*
