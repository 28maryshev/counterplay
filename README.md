# Counterplay — тестовый драфт (веб-прототип)

Песочница для отработки драфт-интерфейса и движка рекомендаций **без**
подключения к клиенту LoL. Ты сам расставляешь пики союзников и врагов, а
программа в реальном времени предлагает, кого взять на твою роль, с
обоснованием (база / контра против оппонента / контра против прочих / синергия).

## Запуск
Сборка и зависимости не нужны. Два способа:
1. Открыть `index.html` двойным кликом в браузере.
2. В VS Code — расширение **Live Server** → правой кнопкой по `index.html` →
   «Open with Live Server». Так подтянется актуальная версия иконок Data Dragon.

(Иконки грузятся как картинки напрямую с Data Dragon — работают и без Live Server;
если какая-то не загрузится, на её месте будут инициалы чемпиона.)

## Как пользоваться
- Сверху выбери **свою роль** (по умолчанию Support).
- Кликни слот: слева — союзник, справа — враг. Затем кликни чемпиона в сетке
  снизу — он встанет в выбранный слот, выбор перейдёт к следующему пустому.
- В центре — топ-6 рекомендаций для твоей роли. Клик по карточке «локает»
  чемпиона как твой пик.
- Кнопка **Бан** включает режим банов: клики по сетке банят / снимают бан.
- **Сброс** очищает доску.

## Что показывают цифры
`score` — это **относительная** оценка (сумма дельт), а не винрейт. Чем выше —
тем лучше пик в текущем составе. Полоски под именем — вклад каждого фактора:
- **база** — базовый винрейт чемпиона;
- **vs опп.** — контра против прямого оппонента на линии (максимальный вес);
- **vs врагов** — средняя контра против остальных врагов;
- **синергия** — средняя синергия с союзниками.

## Важно
Все винрейты, контр-матчапы и синергия — **синтетические заглушки**
(детерминированно сгенерированы из id чемпионов), чтобы видеть работу механики.
Реальные числа появятся, когда подключим пайплайн MATCH-V5 (этап 3). Формула
скоринга — ровно как в плане проекта.

## Файлы
- `index.html` — разметка.
- `styles.css` — оформление под драфт LoL (тёмно-синий + hextech-золото).
- `engine.js` — чемпионы, генерация данных, скоринг и обоснование (работает и в Node).
- `app.js` — интерфейс: состояние, рендер, иконки.
